## 一个场景，看懂区别

你打开 ChatGPT，说："帮我规划一次5天的日本旅行。"

它给你一个行程表：第一天东京塔，第二天浅草寺，第三天坐新干线去京都……挺详细的，但也就到这儿了。**它给了你建议，剩下的事全得你自己干**——查机票、比价格、订酒店、加日历，一步步手动操作。

现在换一个方式。你把同样的需求交给一个 AI Agent。

它会做什么？自己去搜航班，对比价格和时间，挑出最优方案；然后去订酒店，选离地铁站近的；最后把整个行程塞进你的日历，发一封确认邮件给你。**全程你只需要说一句"帮我搞定"。**

这就是 Chatbot 和 Agent 的区别。一个只会说，一个真的会做。

## Chatbot vs Agent：到底差在哪

用一张表说清楚：

| 维度 | 普通聊天 AI (Chatbot) | AI Agent |
|------|---------------------|----------|
| 工作方式 | 你问一句，它答一句 | 自己拆解步骤，逐步执行 |
| 能力范围 | 生成文字回答 | 调用工具、访问网络、执行操作 |
| 主动性 | 被动等你提问 | 主动规划和行动 |
| 终点 | 给你一个建议 | 帮你把事情做完 |

一句话总结：**Chatbot 是顾问，Agent 是执行者。**

你问 Chatbot "怎么去上海"，它告诉你"建议坐飞机，国航 CA1234 早上8点起飞"。你问 Agent 同样的问题，它直接帮你把票订了，值机选了，出租车也约好了。

这不是程度的差别，是**本质的差别**——Agent 多了一双手，能操作、能执行、能闭环。

## Goal：Agent 存在的意义

Agent 不是用来闲聊的。**它存在的唯一目的，就是帮你完成一个目标（Goal）。**

这个目标必须是具体的、可观测的、有明确终点的。

打个比方。你雇了一个实习生，你会怎么说？

- ❌ "帮我看看机票。"——看什么？看到什么时候？看完然后呢？
- ✅ "帮我订好下周三飞上海的机票，经济舱，下午出发，订完把行程单发我邮箱。"

第二种说法，实习生知道该干什么、干到什么程度算完。**给 Agent 定目标也是一样的道理。**

模糊的目标导致混乱的行为。你说"帮我看看机票"，Agent 不知道是该搜一遍就停，还是搜到最便宜的才停，还是搜完还要帮你下单。它可能会反复搜索、不停切换、做了很多事但没有一件是你想要的。

**好的 Goal 有一个检验标准：你能明确判断"完成了"还是"没完成"。** "订好机票"是完成，"看看机票"不是。

## Prompt：给 Agent 的任务书

Goal 定好了，怎么把 Goal 传达给 Agent？靠 **Prompt（提示词）**。

Prompt 就是你给 Agent 的"任务书"。写得好，Agent 干活利索；写得烂，Agent 瞎忙活。

这里分享一个我一直在用的框架，叫 **R-G-B 框架**：

- **R - Role（角色）**：告诉 Agent 它是谁。"你是一个资深旅行规划师，擅长性价比出行方案。"
- **G - Goal（目标）**：告诉 Agent 要做什么。"帮我规划一次5天的日本旅行，总预算控制在1万以内。"
- **B - Behavior（行为）**：告诉 Agent 怎么做。"优先选择直飞航班，每天安排不超过3个景点，午餐推荐当地特色小吃。"

三个要素缺一不可。没有 Role，Agent 的回答可能不专业；没有 Goal，Agent 不知道干什么；没有 Behavior，Agent 可能干出你不想要的事。

**前端同学可以这样理解：Prompt 就像组件的 Props。** 你传什么配置进去，组件就怎么表现。Props 写得越精确，组件行为越符合预期。Prompt 也是一样——你给的上下文和约束越清晰，Agent 的输出就越靠谱。

## 一张图看懂 AI Agent

把上面的内容浓缩成一句话：

![AI Agent 概念总览：LLM + Goal + Tools](/images/agent-concept-overview.png)

```
AI Agent = LLM（大脑） + Goal（目标） + 行动能力（工具）
```

拆开来看：

```
┌─────────────────────────────────┐
│            AI Agent             │
│                                 │
│  🧠 LLM —— 负责思考和决策       │
│  🎯 Goal —— 明确要做什么         │
│  🔧 Tools —— 负责执行操作        │
│     ├─ 搜索引擎（查信息）        │
│     ├─ API 接口（调服务）        │
│     ├─ 代码执行（跑脚本）        │
│     └─ 数据库（读写数据）        │
└─────────────────────────────────┘
```

**LLM 是大脑**，负责理解你的需求、拆解任务、做决策。**Goal 是方向**，确保 Agent 知道往哪走。**Tools 是手脚**，让 Agent 能真正操作外部世界。

三者缺一对一个**实用的 Agent** 来说都不完整。光有大脑没有工具，那还是 Chatbot；光有工具没有目标，那只是一个自动化脚本。当然，这是最基础的模型，后面我们还会往上加更多层——记忆、反思、协作、安全护栏，一层一层把 Agent 武装起来。

## 一个常见误区

**"Agent 是不是有自由意志的 AI？它会自己乱来吗？"**

这是很多人刚接触 Agent 时会有的担忧。

答案是：**Agent 不会自己设定目标。** 它所有的"自主行为"，都在你预设的框架内运行。

你看到 Agent 在"自主规划"，看起来很智能。但实际上，它做的事情是：

1. 你给了一个 Goal
2. 它把 Goal 拆成若干步骤
3. 逐步执行，每一步检查结果
4. 如果某一步失败了，换一种方式重试
5. 直到 Goal 完成，或者超过你设的限制（比如最大重试次数）

**这个循环的每一环，都在你定义的规则之内。** Goal 是你设的，工具是你给的，权限是你定的，停止条件也是你写的。Agent 只是在约束范围内执行，不是真的"自由意志"。

我自己写 Agent 框架的时候，这个循环的核心就是一个 `execute_loop()` 方法——一个 `while True`，每轮让 LLM 决定下一步，调工具拿结果，再判断该继续还是该停。看起来的"自主"，说白了就是一个**带权限检查的循环**，跟你写的 `while` 没什么本质区别。

## 写在最后

回顾一下今天的核心收获：

- **Agent ≠ Chatbot。** 关键区别在于：Agent 有目标、有工具、能执行，Chatbot 只能聊天。
- **Goal 是灵魂。** 没有明确目标的 Agent，就像一个没有方向的实习生，忙但无效。
- **Prompt 是任务书。** R-G-B 框架（Role-Goal-Behavior）帮你写出靠谱的 Prompt。

说实话，刚开始学 Agent 的时候，我也觉得这东西很神秘。网上各种文章把它说得像科幻电影里的 AI，要么无所不能，要么马上失控。但真正动手学了底层原理之后发现——**它就是一个有大脑、有目标、有工具的程序，没那么玄乎。**

理解了 Goal 和 Prompt 这两个概念，你已经跨过了 AI Agent 最基础的门槛。
